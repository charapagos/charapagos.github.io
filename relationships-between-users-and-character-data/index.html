Relationships between users and character data

2017-05-18

Relationships between users and character data are explained by the following example.

  • User A

  • Character data “a” that User A owns

  • User B

  • Character data “b” that User B owns

  • User C

  • Character data “c” that User C owns

  • User A and User B are good friends and have already registered each other’s ID in their own friend list.

  • User B and User C are good friends and have already registered each other’s ID in their own friend list.

  • User A and User C do not know each other and do not register each other’s ID in their own friend list.

User A can use character data “b” that User B owns to enjoy online Cinematic Communication with User B. User B can use character data “a” that User A owns to enjoy online Cinematic Communication with User A likewise.

User B can use character data “c” that User C owns to enjoy online Cinematic Communication with User C. User C can use character data “b” that User B owns to enjoy online Cinematic Communication with User B likewise.

User A cannot use character data “c” that User C owns to enjoy online Cinematic Communication with User C. User C cannot use character data “a” that User A owns to enjoy online Cinematic Communication with User A likewise.

It is necessary for User A to register User C’s ID in his/her friend list in advance and to be registered his/her ID in User C’s friend list in advance in order to use character data “c” that User C owns.

It means that they can use character data that their communication partner owns only when their ID has already been registered mutually in their friend list.

Even User A’s ID and User C’s ID have already been registered on the User B’s friend list, User B cannot use character data “a” that User A owns when User B enjoys Cinematic Communication with User C. User B can use character data “a” only when User B enjoys Cinematic Communication with User A.
